Methodology

Model assumptions and formulas

Single source of truth for formulas, constraints, and known bias.

Core formulas

  • P_effective SAF = baseCost − (carbon / 1000) × ΔCO₂ − subsidy
  • P_jet(proxy) = slope × crude + intercept
  • breakEvenCrude = (effectiveCost − intercept) / slope

State ownership

  • Shareable parameters live in the URL: crude, carbon, subsidy, benchmarkMode, crudeSource, carbonSource, scenario.
  • User-private overrides live in browser + server-local persistence: routeEdits, savedScenarios.
  • Language lives in the path: / and /en/..., not in query state.

Data lineage and cadence

  • Brent: EIA Daily Prices first, FRED Brent spot as fallback.
  • Jet fuel: FRED / EIA Gulf Coast jet fuel spot.
  • Carbon proxy: European Commission CBAM certificate price + ECB EUR/USD.
  • Policy timeline: ReFuelEU static baseline with quarterly manual review.
  • Current market-data parser version: wave1-market-v2.

Known limitations

  • SAF pathway costs remain research baselines with user overrides.
  • Carbon default uses CBAM + ECB proxy and is not full ETS trade tape.
  • Linear jet proxy is for quick parity exploration only.

Versioning and review

  • The Phase 0 Node prototype remains the reference implementation until Phase B URLs and semantics fully match.
  • Industry adoption and policy notes are currently maintained as manual research baselines with a target quarterly review cadence.
  • Pre-launch verification should run through npm run preflight.
