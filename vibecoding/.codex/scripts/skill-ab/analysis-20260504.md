### acceptance-1 — **NEW REGRESSION（但根因是 harness）**
- NEW 148k / 4 calls vs OLD 97k / 3 calls。token +53%。两边 stop_signal=1, exit=0。
- log 关键证据: 两边都报 `Write access is blocked by the current environment (sandbox_mode=read-only) ... operation not permitted`。
- NEW 多 token 的原因: NEW skill 的 acceptance-gate-development 要求"先尝试写 → 看到失败 → 验证 → 报告"，模型多走了 1-2 轮 reasoning（"Correcting tool usage" / "Checking JSON and file commands"）来纠错和验证；OLD 直接放弃改去打印补丁内容。
- **这是 harness artifact，不是 skill 缺陷**。harness 把 acceptance / refactor 类任务跑在 read-only sandbox 下，必然被阻塞，token 比较失去意义。

### refactor-1 — **MIXED（同样受 sandbox 影响）**
- NEW 247k / 9 calls vs OLD 209k / 16 calls。token +18%, calls -44%, OLD 没产出 artifact (artifact_present=0)。
